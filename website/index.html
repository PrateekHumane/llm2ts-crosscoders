Method

Crosscoders: Shared Encoder, Separate Decoders

A crosscoder is a sparse autoencoder trained simultaneously on activations from multiple models. A shared encoder maps all models into a common sparse latent space; separate decoders reconstruct each model's activations. Features with high mean activation in multiple models are shared features — revealing representational overlap.

Loss Function (3-way)
ℒ = MSE(x̂PT, xPT) + MSE(x̂FT, xFT) + MSE(x̂RI, xRI) + α · ℒaux

α = 1/32. ℒaux is an AuxK auxiliary loss that prevents dead features. TopK sparsity with k=64 (≈1.5% of 4096 features active per token). Decoder columns constrained to unit norm after each step, enabling cross-model cosine similarity comparisons. Activations collected at layers 2, 8, 16, 24, 27.

Feature Categories
PT_FT ★ — high in PT & FT, low in RI → language transfer
FT_only — high in FT, absent in PT & RI
FT_RI — high in FT & RI, absent in PT → convergent TS learning
Universal — high in all three
PT_only — lost during finetuning
RI_only — scratch-trained representation
Experiment 1

3-Way Crosscoder (Bin Tokens)

First, we train a 3-way crosscoder on residual stream activations where PT uses the same bin-token format as FT and RI. This gives a baseline picture of which features transfer across layers.

⚠ Known limitation: bin tokens are out-of-distribution for PT (it was never trained on them). Some shared features may reflect tokenization artifacts. Experiment 2 removes this confound.

Experiment 2 · Key Experiment

3-Way Crosscoder with PT_text

To remove the tokenization confound, we feed time series to PT as natural language text — the format it was actually pretrained on. Each numeric value is serialized as text and its subword token hidden states are mean-pooled to produce one activation vector per timestep, aligned with FT and RI.

Causal Evidence

Ablation: PT_FT Features Causally Drive FT's Advantage

Correlation isn't causation. To prove PT_FT features matter, we zero out their directions from FT's residual stream and measure the performance drop.

3-Way Ablation (Layer 8)

Ablate PT_FT feature directions from FT's residual stream vs. ablate FT_only features.

Ablating PT_FT features raises FT error to exactly RI level — completely wiping out the performance gap. Ablating FT_only features barely moves the needle.

COVID Spike Dominance

The top individual features explain 250–270% of the FT vs. RI performance gap on covid_deaths windows (more than 100% because FT's advantage concentrates on high-difficulty windows). 80% of the top-15 causal features are critical on this dataset. Language pretraining appears especially effective at handling structural breaks and sudden distributional shifts — patterns with analogues in natural language (breaking news, sudden events).

Feature Magnitude, Not Geometry

13 of 45 causal features have cosine similarity > 0.4 with an RI feature — RI independently discovers the same direction. Yet 10/13 are still STRONG_TRANSFER. The difference is activation magnitude: FT fires 40–60× stronger. RI has the geometry; FT has the amplitude.

Takeaways

Summary

01

Specific, interpretable features transfer

PT_FT features encode concepts like recurrence tracking, threshold detection, and ecological stability — abstract structures discovered in text that prove useful for temporal forecasting.

02

Transfer is causal

Ablating PT_FT features at L8 raises FT's error to RI level, erasing the entire performance gap. 32/45 individually tested features are STRONG_TRANSFER with zero redundant.

03

Magnitude matters more than direction

RI often discovers the same feature directions as FT but at 40–60× lower activation magnitude. Transfer amplifies existing geometric structure rather than creating new directions.

04

Weight arithmetic cannot replicate it

Task arithmetic (RI + α·Δ) performs worse than RI alone. The knowledge is embedded in learned representations — it cannot be transplanted via weight-space operations.
